# 需求 · v3.2 驾驶舱冲击力 + 数据飞轮显性化

> 2026-06-01 · 吕志远 + Napoleon · 需求记录稿（先记不做，后续规划再跑）
> 来源：吕博士看完 v3.1 demo 后的反馈。v3.1 已完成（今日驾驶舱/任务池/planner/profiles/prompts），本文是下一轮 v3.2 的方向。

## 核心反馈（两条）

### 需求 1 · 「今天做什么」要有冲击力（最高优先）
**问题**：当前驾驶舱虽然列了今日任务，但"一打开就知道今天要做什么"的冲击力不够——信息是平铺的，大脑没有被一个明确动作"击中"。

**目标**：用户一打开面板，**第一眼、零阅读成本**就被一个明确的「现在就做这一件事」击中，而不是看到一堆卡片再自己挑。

**设计方向（待规划，候选）**：
- **英雄区(Hero)**：首屏顶部放一个超大、聚焦的「今天第一件事」块——大字任务标题 + 一句最小启动动作 + 一个"开始"的视觉锚点，占据视觉重心。
- 其余次任务/保底任务**视觉降级**（缩小、弱化），先不抢注意力。
- 可能的强化手法：大字号、留白、单一主色聚焦、"▶ 现在就做这一步"的行动召唤、把 starter（最小启动）提到最显眼位置（因为"启动"比"目标"更重要）。
- 文案冲击：如「今天只做这一件 → [任务] → 先做这一步：[starter]」。

> 原则呼应任务书 §11：用户打开后不是看到一堆信息，而是立刻知道今天从哪个最小动作开始。v3.1 做到了"列出来"，v3.2 要做到"击中"。

### 需求 2 · 数据飞轮要显性化、更显眼
**问题**：demo 里没看到"数据飞轮"被突出。飞轮（大目标→小动作→启动任务→完成→正反馈→再启动）是这套系统的灵魂理念，但现在用户感知不到它的存在。

**目标**：一打开就**提醒用户"数据飞轮"的存在**，让用户意识到自己正处在飞轮的哪个环节、完成一个小动作就在推动飞轮转。

**设计方向（待规划，候选）**：
- 在驾驶舱显眼处放一个**飞轮可视化**：环形/循环示意（启动→行动→完成→反馈→再启动），高亮当前所处环节。
- 把"最近完成(正反馈)"和"今日启动"在视觉上连成一个**循环闭环**，而不是两个独立列表——让用户看到"我完成的 → 喂回飞轮 → 推动今天的"。
- 顶部一句话点题飞轮理念（如"科研飞轮：每个小动作都在让它转起来"）。
- 可考虑一个轻量"飞轮转动"动效/进度感，强化"动起来了"的反馈。
