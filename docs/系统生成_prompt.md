# 科研流水线 · 系统生成 Prompt (v2 · 2026-05-30)

> 用法：把下面【===】之间的整段给任何 coding agent，它就能从零重建整个系统。
> 这是自包含的系统规格 prompt，已对齐当前实现（三级下钻 + 文档库三分类 + README + 日夜模式）。

```
=== 科研流水线 (Research Pipeline) · 系统构建规格 v2 ===

# 你的任务
用纯 Python 标准库（零第三方依赖）构建一个本地动态网页应用「科研流水线」，
帮科研人员管理论文/专利/项目从想法到发表的全流程。一条命令 `python3 app.py`
启动，浏览器访问 localhost。要求自包含、可打包成独立 exe/app 发布，
绝不依赖 Notion 或任何外部软件/账号。

# 产品定位
- 名称：科研流水线 (Research Pipeline)
- 核心理念：【流程/状态驱动】，不是文件浏览器。进入一个项目先看它"在流程的
  哪个状态"，文件是末端内容，不是主角。
- 用户：先自用（博士生 + 硕士生），后开源发布。

# 术语（统一用这套词）
| 概念 | 定名 | 含义 |
|------|------|------|
| 产品 | 科研流水线 | 整个应用 |
| 分组层 | 类别 | 硕博论文 / 小论文 / 专利 / 项目报告 / 其他 |
| 流程单元 | 状态 | 4 块：选题立项 → 实验执行 → 成文 → 投稿发表 |
| 流程子单元 | 子步骤 | 每个状态下 2-3 个子步骤（共 10 个，见下）|
| 视图1 | 状态总览 | 看项目当前在哪个状态（主视图）|
| 视图2 | 文档库 | 浏览/查看项目文档 |
对用户只暴露上面的中文术语；substep/block/__all__ 等是内部标识，不外泄到界面。

# 4 状态 × 10 子步骤（每个子步骤带"标准模板要点" tpl）
1. 选题立项 = Idea + 验证创新性 + 设计实验
2. 实验执行 = 跑实验 + 评估可行性          （注意：用"评估可行性"，与磁盘目录一致）
3. 成文     = 选期刊 + 写稿 + 审稿
4. 投稿发表 = 投稿 + 跟进修回
每个子步骤三态：todo(未开始) / doing(进行中) / done(已完成)，配色 灰/绿/蓝。

# 数据模型（本地文件即真源，无数据库）
- 项目源目录：每个项目 = 一个文件夹。文件夹名前缀决定类别：
  thesis*/phd*/含"博论" → 硕博论文；paper_* → 小论文；patent* → 专利；
  projectbook*/report* → 项目报告；其他 → 其他。
- 项目文件夹内：versions/ledger.json（版本日志，可选）+ 任意 md/docx/html/... 文件。
  约定 stages/<序号_名>/standard.md 放各子步骤的标准格式文档。
- 状态目录：每个项目一个 <项目名>.state.json：
  {
    "title_cn": "界面显示名（无则回退文件夹名）",
    "summary": "一句话概况",
    "current": "选题立项",
    "blocks": {"选题立项":"doing","实验执行":"todo","成文":"todo","投稿发表":"todo"},
    "substeps": {"Idea":"done","验证创新性":"doing", ...},   // 子步骤→状态
    "substep_docs": {"Idea":"stages/1_idea/standard.md", ...}, // 子步骤→md相对项目根路径
    "note": "当前状态备注"
  }
  缺 state.json 的项目：current=选题立项，blocks 全 todo，substeps/substep_docs 为空，优雅降级。

# 后端（Python http.server, stdlib）
- 路径不要硬编码绝对路径！用 os.path.dirname(os.path.abspath(__file__)) 定位，
  STATE_DIR=<repo>/data，PROJ_ROOT 默认指向仓库内 demo 项目目录，
  允许环境变量 RP_PROJ_ROOT / RP_PORT 覆盖；PyInstaller 下用 sys._MEIPASS 兜底。
- 端口默认 8771，被占用要 try/except OSError 并提示换端口（别直接崩）。
- GET /            → 内嵌 index HTML
- GET /api/tree    → {projects:[...], flow:[4状态定义,每个含 subs:[{name,tpl}]]}
  每个 project：{name, line(类别), title_cn, versions, last_why, last_date, state, files:[{rel,path,mtime,ext}]}
- GET /api/file?path=<相对 PROJ_ROOT 的路径> → 文本类(md/txt/py/json/csv/tex)返回内容；
  其他(docx/pdf...)返回 {type:"binary", note:"双击本地查看"}。必须路径越界防护(realpath 前缀校验)。
- 扫描排除 .git / versions / node_modules / 隐藏文件。

# 前端（HTML+JS 内嵌在 app.py 字符串，无构建步骤）
布局：左侧栏(270px) + 主区。
- 左侧栏：标题"🔬 科研流水线"；顶部一个【📖 功能介绍(读我)】默认页入口（进 app 先看它，
  讲清产品用途/三级怎么点/两个视图/刷新）；下面项目按【类别】分组列出；
  底部"＋新建项目""＋新建工作台"(灰度占位) + 【☀️日间/🌙夜间】切换按钮(localStorage 记忆)。
- 主区顶：项目名 + "当前:<状态>" + 版本号 + 实时同步指示点；下面两个 tab：
  【📋 状态总览】(默认) 和 【📂 文档库】。

## 状态总览 = 三级下钻
- 一级：横向流程条，4 个状态块从左到右 → 连接，doing 绿框高亮/done 蓝/todo 灰，
  每块列出其子步骤(带色点)；下方项目概况(summary)+备注(note)。
- 点一个状态 → 二级：该状态的子步骤【竖排卡片】，每张卡片含状态徽章 + "标准模板要点"(tpl)。
- 点一个子步骤 → 三级：渲染该子步骤的 standard.md（substep_docs 指向）；没有则提示"待建"。
- 面包屑可逐级返回（状态总览 › 状态 › 子步骤）。

## 文档库 = 三个横向大分类标签
- 【归纳整理版】：左侧纵向目录 = 📄全文档(置顶默认选中) + 4 状态可折叠分组(▾/▸)，
  每组列其子步骤文档(无文档标"待建")；右侧查看器。
  - 全文档 = 聚合视图：项目概况 + 把所有 substep_docs 的 md 按 状态顺序 拼成一篇，
    带"〔状态〕子步骤"分隔。
- 【📎 原始素材】：列扩展名非文本类(docx/pdf/html…)的文件，跟整理版平级。
- 【⚙ 模板·设置】：灰度占位(未上线)，以后给每个状态指定用哪份标准模板。

## 刷新策略
- 顶部【🔄 刷新】手动刷新；同时每 5 秒静默轮询 /api/tree。
- 静默刷新时若正在看三级文档或文档库，跳过重渲染，不打断阅读（手动刷新才强制重画）。

## 配色 / 主题
- 深色为默认(深蓝 #0f1420 底, #4493f8 主蓝, #3fb950 进行绿, #58a6ff 完成蓝)。
- 日间模式：用 body.light 覆盖层只改配色不动布局(浅底深字)，按钮切换 + localStorage 记忆。
- 视觉细节：卡片圆角+轻阴影+hover 微抬升，自定义滚动条，过渡动画，留白舒展。
- onclick 传项目名/状态名等参数必须转义(防含特殊字符时"点了没反应")。

# 功能范围
MVP：只读浏览（选项目→看流程三级下钻→文档库看文档→自动刷新→切日夜）。
后置 v2：新建项目/工作台；界面内编辑写回 md；模板·设置具体功能；
  大论文专属模板(≠小论文)；PyInstaller 打包发布。

# 非目标
- 不依赖 Notion / 外部软件 / 账号；不做云端/多用户；MVP 不做编辑。

# 交付
单个 app.py(后端+内嵌前端) + data/(state.json) + 复用项目文件夹。
启动：python3 app.py → http://localhost:8771

=== 规格结束 ===
```
